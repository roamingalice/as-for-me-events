ASify 年中一番賞

這份頁面是給門市、電商、行銷與營運確認用。活動目標是用官網 LINE 抽券把客人導向門市，再用 NT$1,500 實體一番賞製造到店事件感，同時協助出清指定品項並拉高客單。

活動主目標
拉門市人流、清庫存、拉高客單
門市一番賞
70 籤 / 每籤 NT$1,500 / 每籤必中
官網抽券
LINE 抽獎、發唯一折扣碼、門市核銷贈小禮

活動架構

官網與門市各自有任務：官網負責發券與導流，門市負責成交、抽賞、核銷與兌獎。折扣不過度疊加，維持規則簡單。

官網：LINE 年中抽券

官網滿額後進 LINE 抽獎頁，抽到唯一 SHOPLINE 折扣碼。折扣碼可在官網使用；若到門市使用，門市同事核銷後贈送小禮。

門市：NT$1,500 一番賞

純買籤，每籤必中，不抵購物金。現場以 Pad 呈現抽籤、獎項板與兌換狀態，提升體驗感並降低人工錯誤。

出清策略

五分褲、背心、上衣與指定 Bratop 做成中高階獎項，讓清庫存不被感覺成清倉，而是年中慶限定福利。

官網抽券流程

LINE 作為主要抽獎與核銷載具，不再設計 Email 備援。門市加碼只保留小禮，不再多折 NT$100。

1

官網下單達門檻

訂單完成頁與通知訊息導向 LINE 抽獎頁。門檻與抽獎次數待確認。

2

LINE 綁定抽獎

使用 LINE user id 綁定抽獎紀錄，避免同一資格重複抽取。

3

發唯一折扣碼

系統從對應獎級的折扣碼池發出一組未使用碼，顯示於 LINE 券面。

4

官網或門市使用

官網輸入折扣碼；門市出示 LINE 券面，店員於 POS 點選對應券並核銷贈小禮。

SHOPLINE 折扣碼機制

建議由 ASify 先在 SHOPLINE 建立各獎級活動與唯一折扣碼，再交給抽獎網站依抽中獎級分配。

CSV 欄位建議

每一組 code 只發一次，且 SHOPLINE 後台設定限用一次。抽獎網站只負責「從未發放清單取碼」。

操作原則

每個獎級建立自己的 code pool，例如 A500、B300、C200、D150。抽獎頁抽到 A 券時，只能從 A500 的未發碼清單取一組。這樣既能防轉傳，也方便活動後比對實際使用率。

對外承諾

NT$1,500

每籤必中，最低帶走等值 ASify 商品組。一番賞金額不可折抵購物金，不可兌換現金。

現場限制

服飾獎項皆以門市現場可選尺寸、顏色為準；無法事後寄送、調貨或保留。規則需在 Pad 與現場告示清楚露出。

獎項公開

Pad 頁面需顯示總籤數、已抽籤數、剩餘獎項與 Last One 狀態。每次抽完立即更新，減少認知落差。

Pad 無人 POS 抽籤

門市不一定要做紙籤；Pad 可以讓客人自己點籤，同時保留店員控管付款與兌獎。

推薦流程

客人先在 POS 購買「年中一番賞 NT$1,500」，取得小票或抽獎碼後，到 Pad 點選籤格抽獎。

權限分工

客人負責點籤；店員負責確認付款、按下兌換完成、交付獎品。這樣有互動，也有營運控管。

1

POS 收款

建立商品「年中一番賞」，售價 NT$1,500，不可套用購物金或折扣券。

2

輸入抽獎碼

Pad 輸入交易碼或掃小票 QR，確認該筆付款尚未抽過。

3

客人點籤

畫面翻牌顯示獎項，系統鎖定該籤，不可重抽。

4

店員兌換

店員交付獎品，按「已兌換」，獎項板與紀錄同步更新。

核銷 SOP

門市同事只需要辨識券面、點 POS、完成核銷。越少人工判斷，越不容易出錯。

LINE 官網券核銷

  • 客人出示 LINE 券面與 QR code。
  • 店員確認券狀態為「未使用」。
  • POS 點選對應年中券折扣。
  • 交付小禮，按下「已核銷」。

門市一番賞兌換

  • 確認 POS 已收 NT$1,500。
  • Pad 完成抽籤並顯示獎項。
  • 服飾獎項依現場尺寸顏色挑選。
  • 店員按「已兌換」並更新獎項狀態。

不可做

  • 一番賞金額不可抵購物金。
  • 一番賞不可使用官網折扣碼。
  • 獎品不可換現金。
  • 服飾獎項不事後寄送、不調貨。

待確認事項

這些是開發與門市 SOP 前需要先定案的部分。只要這份清單確認，下一步就能做活動頁與 Pad 原型。

官網門檻

滿多少可抽一次？每筆訂單最多幾抽？是否排除特價品或指定品類？

折扣券級距與數量

A/B/C/D 券的張數、使用期限、門檻、是否限本人或限會員帳號使用。

小禮內容

官網券到門市核銷時贈送的小禮品項、數量與是否限量。

Pad 技術範圍

第一版是否只做抽籤與核銷紀錄，或要串接 SHOPLINE POS / API。

現場庫存

服飾獎項的尺寸、顏色、數量需先盤點，並放在門市可直接兌換的位置。

法規與公告

活動需清楚揭露總籤數、獎項數量、剩餘狀態、兌換限制與不可折現規則。

建議下一步

先讓同事確認規則能不能跑，再進入設計與技術製作。

1. 定案獎項與庫存

確認 70 籤每一賞的可供數量，尤其是褲子、Bratop、五分褲、背心、上衣的實際現貨。

2. 定案 SHOPLINE 規則

建立折扣碼活動、CSV code pool、POS 快捷折扣與「年中一番賞 NT$1,500」商品。

3. 做 LINE 與 Pad 原型

先做可內測版本：LINE 抽券頁、券面、門市核銷頁、Pad 一番賞抽籤頁。
